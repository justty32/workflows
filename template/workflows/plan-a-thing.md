# plan-a-thing — 規劃一件事（單檔工作流）

← [WORKFLOWS](../WORKFLOWS.md)｜[INDEX](../INDEX.md)

規劃活動、旅行、流程、搬家、任意**非開發專案**時的工作流。

> 〔模板說明〕本檔屬「非開發 / 知識工作」flavor。用不到就刪、並從派發表移除。單一計畫變複雜（多子計畫、長期追蹤）就照 [DEV-GUIDE](../DEV-GUIDE.md) 升級成資料夾型。

## 流程

```
Done when: {{目標與範圍明確、關鍵步驟與依賴列出、時間點/負責人定了、卡點已記錄}}
```

```
釐清目標與成功樣貌 → 定範圍（含「不做什麼」）
  → 拆步驟與依賴 → 標時間點 / 負責人
  → 執行 / 追蹤 → 收尾與回顧
```

- 一開始就寫「不做什麼」，避免範圍無限膨脹。
- 要在選項間抉擇 → 用 [decide](decide.md) 工作流。
- 要使用者親自做 / 付錢 / 決定的 → 記到 [WAIT_USER](../WAIT_USER.md)。
- 進行中狀態記到 [SESSION-LOG](../SESSION-LOG.md)，不只留在聊天裡。

## 何時不用

- 三兩步的小事，直接做。
- 還只是模糊想法 → 先記著（idea / roadmap），不急著規劃。
