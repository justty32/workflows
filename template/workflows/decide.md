# decide — 結構化決策（單檔工作流）

← [WORKFLOWS](../WORKFLOWS.md)｜[INDEX](../INDEX.md)

在幾個選項間做選擇、且想留下**可回溯的理由**時使用。

> 〔模板說明〕本檔屬「非開發 / 知識工作」flavor。用不到就刪、並從派發表移除。決策累積成庫就升級成資料夾型（`decide/` 下按主題存決策記錄）。

## 流程

```
Done when: {{選項列齊、準則定義、各選項評估、給出結論與理由、記錄未選原因}}
```

1. 列出真實可行的選項（含「不做」）。
2. 定準則與相對重要性；準則來自**使用者的實際約束**。
3. 各選項對每個準則評估，標明不確定處。
4. 給明確結論與主要理由。
5. 記錄為什麼不選其他，供日後回顧。

## 決策記錄模板

```md
# 決策：{{主題}}
## 結論
## 選項與評估
| 選項 | 優點 | 缺點 | 對關鍵準則 |
|------|------|------|-----------|
## 主要理由
## 未選原因
## 前提（前提改變要重看）
```

- 使用者的實際約束（預算、時間、風險承受）優先於一般最佳解。
- 不確定的地方標出來，不假裝確定。
