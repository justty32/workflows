# organize — 整理資訊結構（單檔工作流）

← [WORKFLOWS](../WORKFLOWS.md)｜[INDEX](../INDEX.md)

把一堆資訊、檔案、筆記整理成好導航的結構。這是 [DEV-GUIDE](../DEV-GUIDE.md) 結構整理原則的**日常應用**（非開發版）。

> 〔模板說明〕本檔屬「非開發 / 知識工作」flavor。用不到就刪、並從派發表移除。organize 主要是「動手整理」，規則本身很短，多半停在單檔。

## 觸發（同 DEV-GUIDE）

- **單體過大**：某檔或某筆記大到難讀。
- **資料夾雜亂**：不同用途混在一起，難找。

## 流程

```
Done when: {{重新分好類、頂層有導航（README/INDEX）、原意未變、導航 index 已更新}}
```

1. 按內容 / 用途 / 主題語意分，不硬切等分。
2. 放進專屬子資料夾或既有主題檔。
3. 子資料夾頂層留 README / INDEX 導航。
4. 原位置只留精簡指標與連結。

- **只搬不改內容**，保留原意；要改內容是另一件工作。
- 一次只動一個面向（先分類、或先改寫，不混做）。
- 材料多到難定位時，建一份**導航 index**（非程式版的 code map，如 `INFO_MAP.md`：哪份材料在哪、負責什麼）。維護鏈精神同 [conventions](common/conventions.md)：材料 → 導航 index → 衍生呈現。
