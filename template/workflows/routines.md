# routines — 例行事務 / 定期待辦

← [INDEX](../INDEX.md)｜[WORKFLOWS](../WORKFLOWS.md)｜由 [tick](tick.md) 每次心跳執行

**做什麼**：記錄「在某個**時機 / 週期**下該做的事」（＝**常規事務**）——上班時、下班前、會議前後、每週、每隔幾天……。這些**不常變動、固定循環**（與 [schedule](schedule.md) 的臨時一次性「行程」相對）。目的有二：① 怕忘記；② 能交給 **agent 代勞**的就標出來。

**兩種進入方式（同一個檔、兩個目的）**：
- **A. 登記 / 維護**（你說「這是常規事務，**幫我加進 routine**」時）→ agent 把它寫進下面的「時機分區」或「間隔登記表」。
- **B. 執行**（[tick](tick.md) 每次心跳叫起時）→ **看現在時間 → 對照下面清單 → 有到期的就做**（下面「執行」段）。

**怎麼組織**：依**觸發時機**分區（下面「時機分區」）＋**間隔型登記表**。每個項目標三件事：**執行者**（我 / agent）、**內容**、（間隔型才要）**上次執行日期**。

> 〔模板說明〕本檔就是 routines 的**清單 + 執行規則**，通用骨架。你只要填自己的**時機分區**與**間隔登記表**＋全域 `{{時區}}` 佔位符。用不到就整檔刪（tick 那步跳過即可）。

## B. 執行（tick 每次叫起就照這做）

> A. 登記 / 維護＝把常規事務寫進下面的「時機分區」或「間隔型登記表」（agent 依你口述增刪）。

1. **判當前當地時間**（`{{時區，如 Asia/Taipei}}`）：幾月幾號、星期幾、幾點幾分。依平台取——
   - **Windows**（PowerShell；git-bash 的 `TZ=… date` 可能回 GMT）：
     ```powershell
     [System.TimeZoneInfo]::ConvertTimeBySystemTimeZoneId((Get-Date), '{{時區 ID，如 Taipei Standard Time}}').ToString('yyyy-MM-dd dddd HH:mm')
     ```
   - **Linux / macOS**（IANA 時區名）：
     ```bash
     TZ='{{時區，如 Asia/Taipei}}' date '+%Y-%m-%d %A %H:%M'
     ```
2. **對照清單**：現在落在下面哪個「時機分區」、有沒有到期的間隔項（距上次執行超過週期）。
3. **到期就做**：
   - **唯讀事務**（盤點、健康檢查）→ 當場做（守 [AGENTS.md](../AGENTS.md) always-on 鐵律）。
   - **會造成不可逆影響**的（動機器檔案 / DB / repo 工作樹）→ **先問使用者**，未確認不動。
   - **重活**（深度巡檢 / 調查）→ **不在這裡跑**，記下 / 之後另開 session。
   - **間隔型做完** → 更新登記表該列的「上次執行」。
4. 沒到期的就略過。（提醒機制**先不做**：到期就當場做 / 當場說，不推外部通知。）

## 時機分區（live 清單，隨時增 / 改 / 刪）

第 2 步判當地時間後對照本區——現在落在哪段就照該段做。

> 〔模板說明〕下面全是**範例分區**，示範一則項目該標哪三件事（執行者 / 內容 /（間隔型才要）上次執行）。刪掉用不到的、換成你自己的時機。

### 每天上班（morning，約 09:00 前）
- （範例）**開工唯讀盤點**（執行者：**agent**，唯讀可代勞）：跑一輪唯讀盤點（例 `{{盤點指令，如 docker ps / 服務健康檢查}}`），發現異常 → 轉對應深查工作流。

### 準備會議（約會議前 15 分）
- （待補：準備會議所需資訊）

### 每天下班（EOD，約 16:30）
- （待補：問我還有沒有事；沒事就跑收尾——保存 / push）

## 間隔型 / 太久沒做

登記表（到期看「上次執行」+「週期」；由第 2 步定時檢）：

| 項目 | 週期 | 執行者 | 上次執行 | 細節 |
|------|------|--------|----------|------|
| （範例）刷新某盤點紀錄 | 每 3 天（或距上次 >3 天）| **agent**（唯讀，可代勞）| — | 到期就跑、更新此格 |
| （範例）問使用者才知道的來歷 | 不定期（使用者有空時）| **使用者口述 → agent 記** | — | 純提醒型，「上次執行」留 `—` |

> 〔模板說明〕上表兩列是**範例格式**：一列靠比對「上次執行 + 週期」判到期（做完更新那格），一列沒硬性週期、純等使用者有空（「上次執行」留 `—`）。換成你自己的間隔型事務；唯讀代跑守鐵律：不碰工作樹、不做不可逆操作。
