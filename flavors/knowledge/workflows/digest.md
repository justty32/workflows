# digest — 讀長文 / 資料做消化（單檔工作流）

← [WORKFLOWS](../WORKFLOWS.md)｜[INDEX](../INDEX.md)

讀長文、報告、影片逐字稿、一堆連結，建立可追蹤的**摘要、索引、出處**。目標是自己看懂並能延續、回訪。

> 〔模板說明〕本檔屬「非開發 / 知識工作」flavor。用不到就刪、並從派發表移除。條目變多、產物成堆時，照 [DEV-GUIDE](../DEV-GUIDE.md) 升級成資料夾型（如 `digest/` 下分 `index.md`、`summaries/`、`themes/`）。

## 流程

```
Done when: {{指定材料讀完、index 更新、摘要落檔、關鍵出處可回溯}}
```

1. 建條目（index），狀態 `⬜/✅/⏸️`。
2. 取得原文，記來源 URL / 標題 / 日期。
3. 自己讀原文或可驗證抽取；**不要只信搜尋摘要或二手轉述**。
4. 產出摘要；跨材料成主題時整理到一起。
5. 更新條目狀態。

## 摘要模板

```md
# {{標題}} 摘要
## 一句話
## 重點
## 對我的用途
## 來源
- {{URL 或位置}}
```

- 原始來源與摘要衝突時，以原始來源為準。

## 何時不用

- 查一個即時事實、不需留存 → 直接查證回答。
- 要動手產出 → 接 write；要行動 → 接 plan-a-thing。
