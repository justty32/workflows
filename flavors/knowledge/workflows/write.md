# write — 內容產出（單檔工作流）

← [WORKFLOWS](../WORKFLOWS.md)｜[INDEX](../INDEX.md)

寫文章、筆記、文件、翻譯、貼文等**給人讀的產出**時的工作流。動筆前先過 [common/writing](common/writing.md)（文風）。

> 〔模板說明〕本檔屬「非開發 / 知識工作」flavor。用不到就整檔刪除、並從 [WORKFLOWS](../WORKFLOWS.md) 派發表移除該列。膨脹了照 [DEV-GUIDE](../DEV-GUIDE.md) 升級成資料夾型。

## 流程

開始前先寫一句完成定義：

```
Done when: {{成品交付、涵蓋指定重點、來源已標註、語氣符合對象}}
```

```
定對象與目的 → outline（大綱骨架）
  → draft（初稿：先完整，再求好）
  → revise（改語氣、結構、事實核對）
  → 標來源 → 交付
```

- 事實、數據、引言核對一手來源並附出處。
- 大改結構前先改 outline，不在初稿裡搬段落。
- 翻譯 / 改寫保留原意；語氣調整不等於竄改內容。
- 跨 session 把未完成段落與待查事實記到 [SESSION-LOG](../SESSION-LOG.md)。

## 何時不用

- 一兩句回覆或即時問答，直接寫。
- 還在釐清要寫什麼 → 先走 digest（讀懂材料）或 plan-a-thing（想清楚）。
